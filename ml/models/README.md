# Model artifacts

Every file in this folder is generated by training/exporting code. This table
groups each artifact by the notebook (and underlying module/script) that produces
it, so you can tell at a glance where a file comes from.

| Producing notebook | Module / script | Artifacts |
|---|---|---|
| `notebooks/model_demo.ipynb` | `src/demo_model.py` | `scam_classifier.joblib` &middot; `metrics.json` &middot; `model_card.json` |
| `notebooks/scam_detection_main.ipynb` (v1 ensemble) | `src/embed_model.py` | `embed_models.joblib` &middot; `embed_metrics.json` &middot; `emb_e5small.npz` |
| `notebooks/scam_detection_main_v2.ipynb` (v2, expanded African corpus) | `src/embed_model.py` (v2 cache/paths) | `embed_models_v2.joblib` &middot; `embed_metrics_v2.json` &middot; `emb_e5small_v2.npz` |
| `scripts/13_export_tfidf_v2.py` (extracts the TF-IDF + LogReg pipeline from `embed_models_v2.joblib`) | — | `scam_tfidf_v2.joblib` **&larr; deployed model** |

## What each kind of file is

- **`*.joblib`** — fitted scikit-learn pipelines/ensembles.
- **`emb_e5small*.npz`** — cached `multilingual-e5-small` sentence embeddings, so notebook re-runs skip re-encoding.
- **`*metrics*.json`** — evaluation results (macro-F1, per-class, per-language) for the corresponding run.
- **`model_card.json`** — summary card for the demo model.

## Deployed model

`scam_tfidf_v2.joblib` is the model in production. It is served by
`ml/serve_v2/app.py` on the Hugging Face Space
`wilsons579-scam-classifier-api-v2.hf.space`, which the mobile app calls
(`mobile/rethicsai/lib/core/config/api_config.dart`). It is embedder-free, so the
container is tiny and there is no cold-start model download.

> Note: files are kept flat (not in subfolders) on purpose — several are loaded by
> fixed relative path in `serve/`, `serve_v2/`, `src/`, and `scripts/13_export_tfidf_v2.py`.
> This README is the grouping; the paths stay stable.
